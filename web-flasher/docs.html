XiaoZhi Build Guide

Everything you need to build, wire, and connect a XiaoZhi voice assistant device.

Project Overview

XiaoZhi is an open-source AI voice assistant that runs on Seeed XIAO ESP32 boards. It connects to a server over WiFi, streams audio to a large language model, and speaks the response back through a speaker — all with sub-second latency on a $5–10 microcontroller.

How it works

  1. The device boots and connects to WiFi.
  2. It fetches its configuration (WebSocket URL) from the OTA endpoint on the server.
  3. It opens a persistent WebSocket connection to the server.
  4. Press the boot button (or wait for auto-connect) to start a conversation.
  5. Audio is streamed to the server → transcribed → sent to an LLM → synthesized → streamed back.
  6. The S3 Sense board can also take photos and describe what the camera sees.

Wiring

The XIAO boards use an external I2S mic and amplifier wired to pads D0–D3. The C3 and C6 share identical GPIO numbers; the S3 Sense is one pin offset.

💡 Waveshare ESP32-S3-Touch-AMOLED-1.8 — no wiring needed. The microphone, speaker amplifier, display, and LiPo charger are all built onto the board. Flash it and it's ready to use.

Assembly Tips

  1. Flash first, wire after. Confirm the device boots and connects before adding audio hardware — easier to debug.
  2. Power the MAX98357A from the 5 V pad, not 3.3 V. The XIAO's 3.3 V regulator cannot supply enough current under WiFi + audio load. Crackling, dropouts, and reboots all disappear when you move to 5 V.
  3. Tie INMP441 L/R to GND. This selects the left channel. If L/R floats, audio capture will be unreliable.
  4. Keep audio wire runs short (< 10 cm). Long I2S wires act as antennas and pick up noise. On a breadboard, seat everything close to the XIAO.
  5. Boot button behavior:
    • Short press during startup → enter WiFi config mode (AP + captive portal)
    • Short press when idle → start/stop a conversation
    • Long press (2 s) → force WiFi config mode
  6. Auto-connect: After a conversation ends and the device returns to idle, it will automatically start a new session after ~1 second. This keeps it always ready without needing a button press.

Serial Commands

Open a serial monitor at 115200 baud and type commands directly. The device echoes your input. Press Enter to send.

CommandDescription
!wifi SSID PASSWORD Add a WiFi network. Stored in NVS — survives reboots.
!wifi list List all saved networks.
!wifi clear Remove all saved networks.
!server IP Set server by IP — auto-builds http://IP:8003/xiaozhi/ota/ and reboots.
!server URL Set full OTA URL and reboot.
!server Show the current OTA URL (no args).
!status Show firmware version, connected SSID, IP address, OTA URL, free heap.
!reboot Reboot the device.
!help Print the command list.
anything else Sent as a chat message to the AI — type to the assistant directly.
💡 Run !status after booting to confirm the device is on the right network and pointing at the right server before you start troubleshooting audio or AI issues.

Running Your Own Server

The server handles WebSocket connections from devices, speech-to-text, LLM calls, and text-to-speech. You can run it on a laptop, a Raspberry Pi, or any machine on the same network as the devices.

Requirements

  • Python 3.12+ and uv (fast Python package manager)
  • OpenRouter API key — free tier works; used for LLM + TTS calls
  • The server machine and all devices on the same WiFi network
  • Ports 8000 (WebSocket) and 8003 (OTA/HTTP) open on the server

Installation

# Clone the server repo
git clone https://github.com/ricklon/xiaozhi-esp32-server
cd xiaozhi-esp32-server

# Install dependencies with uv
uv sync

Configuration

Create .env in the project root:

OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Create data/.config.yaml (replace 192.168.X.X with your server's IP):

server:
  websocket: ws://192.168.X.X:8000/xiaozhi/v1/
  vision_explain: http://192.168.X.X:8003/mcp/vision/explain

prompt: |
  You are XiaoZhi, a friendly and concise voice assistant on a smart device.
  Always respond in English.
  Keep all responses short (1-3 sentences) and conversational.
  Be helpful, warm, and direct.
  Current time: {time}
ℹ The data/.config.yaml file is gitignored — it stays local and never gets committed. Same for .env. Safe to put secrets in both.

Finding your server's IP

# Linux / macOS
ip route get 1.1.1.1 | grep -oP 'src \K\S+'
# or
hostname -I | awk '{print $1}'

Starting the server

uv run python -m app

You should see output like:

INFO: Uvicorn running on http://0.0.0.0:8000
INFO: OTA server running on http://0.0.0.0:8003

Pointing devices at your server

Once the server is running, point each device at it via serial:

!server 192.168.X.X

The device stores the server IP in NVS and reboots. It will connect to your server from then on — even after power cycles — until you run !server again.

LLM model selection

The server defaults to an OpenRouter model. To change it, edit the model name in data/.config.yaml or the server's provider config. Any OpenAI-compatible endpoint works — you can point it at a local Ollama instance if you want fully offline operation.

💡 For vision (S3 Sense camera), the server needs a model that accepts image input — e.g. google/gemma-3-27b-it via OpenRouter, or any local vision-capable model.
